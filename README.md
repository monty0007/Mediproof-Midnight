# MediProof

**HIPAA-compliant healthcare AI dataset provenance on the Midnight Network using Zero-Knowledge proofs.**

Hospitals prove their datasets meet HIPAA requirements mathematically — without ever exposing a single patient record. Researchers train AI models on cryptographically verified, compliant data. Auditors verify everything on-chain.

> For a plain-English explanation of what this project does and why it matters, see [PROJECT_OVERVIEW.md](./PROJECT_OVERVIEW.md).

---

## How It Works

```
Hospital submits dataset metadata
    → commit_dataset ZK circuit runs locally
    → Only hashes hit the Midnight blockchain
    → Record count & identity stay private

Researcher requests training job
    → prove_training ZK circuit runs
    → Model hash committed on-chain
    → Raw data never leaves hospital

Auditor verifies compliance
    → Queries Midnight ledger by dataset ID
    → Proof is cryptographically verifiable
    → No trust required
```

## Architecture

```
Browser (React + Vite :3000)
    │
    ├── /api/*  →  FastAPI backend (:8000)
    │                   │
    │                   ├── asyncpg  →  Azure PostgreSQL  (stores hashes only)
    │                   │
    │                   └── zk_client.py  →  Midnight Service (:6300)
    │                                             │
    │                                             ├── @midnight-ntwrk SDK (WASM)
    │                                             └── Docker Proof Server (:6301)
    │
    └── wallet.js  →  Lace browser extension (Midnight DApp Connector API)
```

### Service responsibilities

| Service | Port | Language | Responsibility |
|---------|------|----------|----------------|
| Frontend | 3000 | React + Vite | Three-role UI: Hospital, Researcher, Auditor |
| Backend | 8000 | Python / FastAPI | REST API, hashing, DB, ZK orchestration |
| Midnight Service | 6300 | Node.js / Express | Midnight SDK bridge (WASM cannot run in Python) |
| Proof Server | 6301 | Docker | Actual ZK circuit execution (groth16) |
| Database | Azure | PostgreSQL 17 | Stores dataset/training/audit records (hashes only) |

## ZK Contract

Two circuits in `contract/src/dataset_proof.compact`:

| Circuit | Proves | Private inputs (never stored) | Public output |
|---------|--------|-------------------------------|---------------|
| `commit_dataset` | Dataset is HIPAA de-identified, ≥500 records, linked to hospital | `dataset_hash`, `record_count`, `hospital_id` | `commitment_hash` on-chain |
| `prove_training` | AI model was trained on the committed dataset | `model_weights_hash`, `dataset_commitment`, `training_rows` | `training_proof` on-chain |

**What goes on-chain:** only the proof hashes and a timestamp.  
**What never leaves the hospital:** patient records, raw counts, hospital identity.

## Database Schema

All tables store **only hashes and metadata** — no patient data ever touches the database.

```sql
datasets          -- dataset_id, schema_hash, disease_category, compliance_proof_hash
                  -- (hospital_name is hashed; record_count is never stored)

training_requests -- request_id, dataset_id, model_output_hash, training_proof_hash

audit_log         -- dataset_id, event_type, actor_hash, proof_hash, timestamp
```

## Quick Start

### 1. Install Python dependencies
```bash
cd mediproof
source .venv/bin/activate          # venv is already inside mediproof/
pip install -r backend/requirements.txt
```

### 2. Install Node dependencies
```bash
npm install
cd frontend && npm install && cd ..
cd midnight-service && npm install && cd ..
```

### 3. Install the Compact compiler
```bash
curl --proto '=https' --tlsv1.2 -LsSf \
  https://github.com/midnightntwrk/compact/releases/latest/download/compact-installer.sh | sh
source ~/.zshrc
compact update 0.30.0
```

### 4. Compile the ZK contract
```bash
npx nightforge compile
# Output: contract/dist/dataset_proof/
```

### 5. Start the Midnight proof server
```bash
docker run -d -p 6301:6300 midnightntwrk/proof-server:latest midnight-proof-server -v
```

### 6. Configure the database
Create `backend/.env`:
```
DATABASE_URL=postgresql://user:password@host:5432/dbname?sslmode=require
```

### 7. Start everything
```bash
source .venv/bin/activate
npm run dev
```

| Service | URL |
|---------|-----|
| Frontend | http://localhost:3000 |
| Backend API | http://localhost:8000 |
| Backend docs | http://localhost:8000/docs |
| Midnight service | http://localhost:6300 |
| Health check | http://localhost:6300/health |

## ZK Modes

The midnight service health endpoint (`GET /health`) reports the current mode:

```json
{
  "zk_mode": "real",
  "contract_compiled": true,
  "proof_server": "reachable"
}
```

| Mode | Condition | Behaviour |
|------|-----------|-----------|
| `real` | Contract compiled + proof server running | Actual ZK circuits via Docker proof server |
| `mock` | Proof server down or contract not compiled | SHA-256 hash used as proof fallback |

## Wallet

Connect Wallet in the navbar uses the [Lace browser extension](https://www.lace.io/) with Midnight enabled. The connector iterates `window.midnight` to find the Midnight DApp Connector by name/rdns and calls `.connect('preprod')`.

Without Lace installed, the app automatically falls back to **demo mode** (generates a mock address) so the full UI is usable locally without a wallet.

**Lace settings for real ZK mode:**
- Network: Preprod
- Proof server: Remote (not local)

## API Reference

### `POST /api/dataset/commit`
Commits a hospital dataset proof to the ledger.

**Request:**
```json
{
  "hospital_name": "General Hospital",
  "record_count": 1200,
  "schema_fields": ["age", "icd_10_code", "outcome_flag"],
  "disease_category": "oncology"
}
```
**Response:** `{ dataset_id, compliance_proof_hash, zk_mode }`

> `hospital_name` is SHA-256 hashed before any processing. `record_count` is used as a private ZK witness and never stored.

---

### `POST /api/training/request`
Submits an AI training job against a proven dataset.

**Request:**
```json
{
  "dataset_id": "...",
  "researcher_id": "researcher@lab.org",
  "model_type": "transformer"
}
```
**Response:** `{ request_id, dataset_id, training_proof_hash, model_output_hash, zk_mode }`

> `researcher_id` is hashed before any processing — the raw ID is never stored.

---

### `GET /api/dataset/list`
Returns all publicly available datasets (no hospital names, no record counts).

### `GET /api/audit/{dataset_id}`
Returns the full public audit record: compliance proof + training proof for a dataset.

### `GET /api/audit/log`
Returns the full event log across all datasets.

## Deployment to Midnight Preprod

To deploy the contract on-chain (optional — not required for local testing):

1. Create a wallet: `npx nightforge wallet create`
2. Fund it at https://faucet.preprod.midnight.network/
3. Update `midnight.config.js` network to `preprod`
4. Run: `node deploy.mjs`

The contract address is saved to `deployment.json`.
